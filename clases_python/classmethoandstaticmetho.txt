@classmethod vs @staticmethod
@staticmethod (Método Estático)
Un método que no accede a la instancia ni a la clase. Es independiente.

class Persona:
    @staticmethod
    def calcularEdad(año_nacimiento):
        return 2026 - año_nacimiento

# Se llama sin crear objeto
print(Persona.calcularEdad(1990))  # 36

Características:

✅ No recibe self ni cls
✅ Se llama desde la clase sin crear instancia
✅ Útil para funciones auxiliares relacionadas con la clase

@classmethod (Método de Clase)
Un método que accede a la clase misma (no a la instancia). Recibe cls en lugar de self.
class Persona:
    cantidad = 0
    
    def __init__(self, nombre):
        self.nombre = nombre
        Persona.cantidad += 1
    
    @classmethod
    def totalPersonas(cls):
        return f"Total de personas: {cls.cantidad}"

p1 = Persona("Juan")
p2 = Persona("María")
print(Persona.totalPersonas())  # Total de personas: 2

Características:

✅ Recibe cls (la clase) como primer parámetro
✅ Accede a atributos de clase, no de instancia
✅ Útil para crear constructores alternativos (factory methods)

class Cuenta:
    def __init__(self, titular, saldo):
        self.__titular = titular
        self.__saldo = saldo
    
    @classmethod
    def cuentaSinSaldo(cls, titular):
        """Constructor alternativo sin saldo inicial"""
        return cls(titular, 0)

# Uso normal
cuenta1 = Cuenta("Juan", 1000)

# Uso con classmethod
cuenta2 = Cuenta.cuentaSinSaldo("María")
Comparativa
Aspecto	@staticmethod	@classmethod	Método normal
Parámetro	ninguno	cls	self
Accede a instancia	❌ No	❌ No	✅ Sí
Accede a clase	❌ No	✅ Sí	✅ Sí
Llamada	Clase.metodo()	Clase.metodo()	objeto.metodo()
Uso	Funciones auxiliares	Constructores alternativos	Lógica de instancia
