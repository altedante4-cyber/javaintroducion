UML BÁSICO (Preguntas 1-10)
Pregunta 1: ¿Qué significa UML?
✓ B) Unified Modeling Language
UML = Unified Modeling Language (Lenguaje Unificado de Modelado)
Pregunta 2: ¿UML es un lenguaje o una metodología?
✓ C) Un lenguaje para modelar
UML es un LENGUAJE, NO una metodología/proceso
Pregunta 3: ¿Quiénes fueron los creadores de UML?
✓ B) Grady Booch, Jim Rumbaugh e Ivar Jacobson
Los 3 padres de UML
Pregunta 4: ¿En qué año OMG adoptó UML 1.1?
✓ B) 1997
Noviembre de 1997
Pregunta 5: ¿Qué versión añadió diagramas de estructura compuesta y timing?
✓ C) UML 2.0
Principios de 2005
Pregunta 6: ¿Qué tipo de diagramas muestran la vista estática?
✓ C) Diagramas estructurales
Los estructurales muestran la vista estática; los de comportamiento, la dinámica
Pregunta 7: ¿Cuál NO es un diagrama de comportamiento?
✓ D) Diagrama de componentes
Diagrama de componentes es estructural
Pregunta 8: ¿Cuál NO se usa para diseñar diagramas UML?
✓ D) Eclipse
draw.io, StarUML y Modelio sí son herramientas UML. Eclipse es un IDE.
Pregunta 9: ¿Qué significa modelar?
✓ B) Crear una representación de la realidad
Modelar = crear una representación de la realidad para analizarla antes de implementar
Pregunta 10: ¿Qué ventaja del modelado menciona la teoría?
✓ B) Facilita la comprensión para nuevos integrantes
Facilita la comprensión, la subcontratación y los cambios
DIAGRAMA DE CLASES (Preguntas 11-20)
Pregunta 11: ¿Cuáles son las partes de un diagrama de clases?
✓ A) Nombre, atributos, métodos
Toda clase tiene: nombre, atributos y operaciones (métodos)
Pregunta 12: ¿Cómo se representa una clase abstracta en UML?
✓ B) Con el nombre en cursiva
Las clases y métodos abstractos aparecen en cursiva
Pregunta 13: ¿Qué símbolo representa visibilidad pública?
✓ B) +
+ = public, - = private, # = protected, ~ = package
Pregunta 14: ¿Qué símbolo representa visibilidad privada?
✓ A) -
- = private
Pregunta 15: ¿Qué símbolo representa visibilidad protected?
✓ C) #
# = protected
Pregunta 16: ¿Cómo se representa un atributo static?
✓ B) Subrayado
Los atributos y métodos static se subrayan
Pregunta 17: ¿Qué estereotipo para enumeración?
✓ B) <>
<> representa tipos enumerados con atributos en mayúsculas
Pregunta 18: ¿Qué estereotipo indica métodos y variables estáticos?
✓ C) <>
<> = clase de utilidad (fecha, concatenación, etc.)
Pregunta 19: ¿Qué es una interfaz en UML?
✓ B) Una especificación de servicios
Interfaz = colección de operaciones que especifican servicios
Pregunta 20: ¿Cuál es la sintaxis correcta de un atributo?
✓ B) [visibilidad] nombre [: tipo]
visibilidad nombre : tipo = valor inicial
RELACIONES (Preguntas 21-30)
Pregunta 21: 'Un coche TIENE UN motor' es...
✓ B) Asociación
Asociación = relación estructural donde los objetos están conectados
Pregunta 22: 'Un corazón ES PARTE DE una persona' es...
✓ C) Composición
Composición = relación parte de con ciclo de vida dependiente (rombo relleno)
Pregunta 23: ¿Cómo se representa la agregación?
✓ B) Rombo vacío
Agregación = rombo vacío (ciclo de vida independiente)
Pregunta 24: ¿Cómo se representa la composición?
✓ B) Rombo relleno
Composición = rombo relleno (ciclo de vida dependiente)
Pregunta 25: 'Una cuenta bancaria DEPENDE de un cliente' es...
✓ D) Dependencia
Dependencia = cambio en especificación afecta a otro pero no viceversa
Pregunta 26: ¿Qué relación indica herencia entre clases?
✓ C) Generalización
Generalización = relación superclase/subclase (herencia)
Pregunta 27: ¿Qué relación indica que una clase IMPLEMENTA otra?
✓ C) Realización
Realización = implementa interfaz (flecha discontinua con triángulo)
Pregunta 28: Diferencia entre agregación y composición
✓ C) La composición tiene ciclo de vida dependiente
Composición = ciclo dependiente (si muere el contenedor, mueren las partes)
Pregunta 29: ¿Cómo se representa la dependencia?
✓ B) Línea discontinua con flecha
Dependencia = flecha discontinua
Pregunta 30: ¿Qué relación usa 'ES UN'?
✓ C) Herencia
Herencia = relación "ES UN" (Perro ES UN Animal)
POO (Preguntas 31-40)
Pregunta 31: ¿Qué es una clase en POO?
✓ B) La descripción de un conjunto de objetos
Clase = molde/plantilla para crear objetos
Pregunta 32: ¿Qué es un objeto en POO?
✓ B) Una instancia de una clase
Objeto = instancia concreta de una clase
Pregunta 33: ¿Se puede instanciar una clase abstracta con 'new'?
✓ B) No, genera error
Las clases abstractas NO se pueden instanciar directamente
Pregunta 34: Diferencia entre clase abstracta e interfaz
✓ B) La clase abstracta puede tener estado, la interfaz no
Clase abstracta = puede tener estado e implementación. Interfaz = solo contrato.
Pregunta 35: ¿Qué modificador permite ver desde cualquier clase?
✓ C) public
public = visible desde cualquier lugar
Pregunta 36: ¿Qué modificador permite ver desde propia clase y subclases?
✓ B) protected
protected = propia clase + subclases + mismo paquete
Pregunta 37: ¿Cuál es el objetivo principal de la POO?
✓ B) Reutilización
La reutilización de código es el objetivo principal
Pregunta 38: ¿Qué indica 'spaguetti code'?
✓ C) Ifs anidados excesivos
Spaguetti code = código con muchos if/else anidados, difícil de mantener
Pregunta 39: ¿Qué relación N a N se resuelve con clase de asociación?
✓ C) Asociación
Las relaciones N a N se resuelven con clase de asociación
Pregunta 40: ¿En qué consiste el modelado ágil?
✓ B) Documentación ligera y justo a tiempo
Modelado ágil = documentación ligera, ordenada, accesible, JIT (justo a tiempo)
