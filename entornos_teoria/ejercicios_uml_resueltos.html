EJERCICIO 1 - Clase Abstracta con Herencia

ENUNCIADO:

Dibuja el diagrama UML de las siguientes clases:

public abstract class Figura {
    protected String color;
    
    public Figura(String color) { ... }
    public abstract double calcularArea();
}

public class Rectangulo extends Figura {
    private double base;
    private double altura;
    
    public Rectangulo(String, double, double) { ... }
    public double calcularArea() { ... }
}

SOLUCIÓN:

Figura
# color: String
+ Figura(color: String)
+ calcularArea(): double
extends
Rectangulo
- base: double
- altura: double
+ Rectangulo(c: String, b: double, a: double)
+ calcularArea(): double

Explicación:

  • Figura es abstracta (nombre en cursiva)
  • calcularArea() es abstracto (cursiva) en Figura
  • Rectangulo hereda de Figura (extends)
  • La flecha con triángulo hueco indica herencia/generalización
  • Rectangulo implementa calcularArea()
EJERCICIO 2 - Composición (Parte de)

ENUNCIADO:

Modela un sistema donde un Coche tiene un Motor. El motor no puede existir sin el coche.

public class Coche {
    private String marca;
    private Motor motor;
    
    public Coche(String marca, Motor motor) { ... }
    public void arrancar() { ... }
}

public class Motor {
    private int cilindrada;
    
    public Motor(int cilindrada) { ... }
}

SOLUCIÓN:

Coche
- marca: String
- motor: Motor
+ Coche(marca: String, motor: Motor)
+ arrancar(): void
1 "tiene un" 1
composición
Motor
- cilindrada: int
+ Motor(cilindrada: int)

Explicación:

  • Composición: El rombo negro indica ciclo de vida dependiente
  • Si el Coche se destruye, el Motor también se destruye
  • Multiplicidad 1..1 (obligatorio) en ambos lados
  • Relación "TIENE UN" o "PARTE DE"
EJERCICIO 3 - Agregación (Tiene un - ciclo independiente)

ENUNCIADO:

Modela una Biblioteca que contiene Libros. Los libros pueden existir independientemente de la biblioteca.

public class Biblioteca {
    private String nombre;
    private ArrayList<Libro> libros;
    
    public Biblioteca(String nombre) { ... }
    public void agregarLibro(Libro libro) { ... }
}

public class Libro {
    private String titulo;
    private String autor;
    
    public Libro(String titulo, String autor) { ... }
}

SOLUCIÓN:

Biblioteca
- nombre: String
- libros: Libro[0..*]
+ Biblioteca(nombre: String)
+ agregarLibro(libro: Libro): void
1 "contiene" 0..*
agregación
Libro
- titulo: String
- autor: String
+ Libro(titulo: String, autor: String)

Explicación:

  • Agregación: El rombo VACÍO indica ciclo de vida independiente
  • Los libros pueden existir sin la biblioteca (venderlos, prestarlos, etc.)
  • Multiplicidad 0..* (0 o más libros)
  • Libro[0..*] indica colección multivaluada
EJERCICIO 4 - Interfaz e Implementación

ENUNCIADO:

Crea una interfaz Volador y una clase Avion que la implemente.

public interface Volador {
    void volar();
    int getAltitude();
}

public class Avion implements Volador {
    private int altitude;
    
    public void volar() { ... }
    public int getAltitude() { ... }
}

SOLUCIÓN:

«interface» Volador
+ volar(): void
+ getAltitude(): int
implements
Avion
- altitude: int
+ volar(): void
+ getAltitude(): int

Explicación:

  • Interfaz: Se marca con «interface»
  • Los métodos de la interfaz son abstractos (cursiva)
  • Realización: Flecha discontinua con triángulo hueco
  • Avion implementa TODOS los métodos de Volador

EJERCICIO 6 - Asociación N a N con Clase de Asociación

    private String nombre;

public class Curso {
    private String nombre;
    private ArrayList<Matricula> matriculas;
}

public class Matricula {
    private String fecha;
    private double nota;
    
    public Matricula(String fecha) { ... }

SOLUCIÓN:

Estudiante
- nombre: String
+ getNombre(): String
Curso
- nombre: String
+ getNombre(): String
Matricula
- fecha: String
- nota: double

0..* "se matricula en" 0..* "tiene"

EJERCICIO 7 - Dependencia (Usa)

ENUNCIADO:

Modela donde una clase Cliente usa temporalmente una clase Calculadora para calcular su IMC.

public class Cliente {
    private String nombre;
    private double peso, altura;
    
    public double calcularIMC() {
        Calculadora calc = new Calculadora();
        return calc.dividir(peso, altura * altura);
    }
}

public class Calculadora {
    public double dividir(double a, double b) {
        return a / b;
    }
}

SOLUCIÓN:

Cliente
- nombre: String
- peso: double
- altura: double
+ calcularIMC(): double
«uses»
Calculadora
+ dividir(a: double, b: double): double

Explicación:

  • Dependencia: Relación TEMPORAL/PUNTUAL (usa)
  • Cliente CREA un objeto de Calculadora dentro de un método
  • Se representa con flecha discontinua
  • Si Calculadora cambia, podría afectar a Cliente, pero no al revés
EJERCICIO 8 - Sistema de Gestión de Biblioteca (Ejercicio Completo)

ENUNCIADO:

Implementa el siguiente diagrama UML:

// Clase abstracta Material
public abstract class Material {
    protected String titulo;
    protected String autor;
    
    public abstract String getTipo();
}

// Clase Libro
public class Libro extends Material {
    private int numPaginas;
    
    public String getTipo() { return "Libro"; }
}

// Interfaz Prestable
public interface Prestable {
    void prestar();
    void devolver();
}

// DVD implementa Prestable
public class DVD extends Material implements Prestable {
    private int duracion;
    
    public String getTipo() { return "DVD"; }
    public void prestar() { ... }
    public void devolver() { ... }
}

SOLUCIÓN:

Material
# titulo: String
# autor: String
+ getTipo(): String
«interface» Prestable
+ prestar(): void
+ devolver(): void
Libro
- numPaginas: int
+ getTipo(): String
DVD
- duracion: int
+ getTipo(): String
+ prestar(): void
+ devolver(): void

Explicación:

  • Material es abstracta (nombre y método en cursiva)
  • Libro hereda de Material (extends)
  • DVD hereda de Material e implementa Prestable (extends + implements)
  • Prestable es interfaz (estereotipo <>)
  • Símbolos: +=public, -=private, #=protected
