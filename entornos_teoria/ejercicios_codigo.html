1. HERENCIA - La palabra clave SUPER
TEORÍA: La palabra super se usa para invocar el constructor o métodos de la clase PADRE. Si hay un constructor en el padre, el hijo DEBE llamarlo con super().
// CLASE PADRE public class Animal { protected String nombre; public Animal(String nombre) { this.nombre = nombre; System.out.println("Constructor de Animal: " + nombre); } public void comer() { System.out.println(nombre + " está comiendo"); } } // CLASE HIJA public class Perro extends Animal { private String raza; public Perro(String nombre, String raza) { super(nombre); // <-- LLAMA al constructor del padre this.raza = raza; System.out.println("Constructor de Perro: " + raza); } @Override public void comer() { super.comer(); // <-- LLAMA al método del padre System.out.println(nombre + " come croquetas"); } } // MAIN public class Main { public static void main(String[] args) { Perro p = new Perro("Rex", "Pastor Alemán"); p.comer(); } }
SALIDA: Constructor de Animal: Rex Constructor de Perro: Pastor Alemán Rex está comiendo Rex come croquetas
EXPLICACIÓN:
1. Se crea el Perro, que llama a super(nombre)
2. Se ejecuta el constructor del padre (Animal)
3. Luego continúa con el constructor del hijo
4. super.comer() ejecuta el método del padre, LUEGO el del hijo
2. POLIMORFISMO - Enlazado Dinámico
TEORÍA: El polimorfismo permite que un objeto de tipo PADRE apunte a un objeto de tipo HIJO. El método que se ejecuta se decide en TIEMPO DE EJECUCIÓN (no de compilación).
public class Animal { public void hacerRuido() { System.out.println("...ruido..."); } } public class Perro extends Animal { @Override public void hacerRuido() { System.out.println("¡Guau!"); } } public class Gato extends Animal { @Override public void hacerRuido() { System.out.println("¡Miau!"); } } public class Main { public static void main(String[] args) { // Variable de tipo Animal, pero objeto Perro Animal a1 = new Perro(); Animal a2 = new Gato(); Animal a3 = new Animal(); a1.hacerRuido(); // ¿Qué imprime? a2.hacerRuido(); // ¿Qué imprime? a3.hacerRuido(); // ¿Qué imprime? } }
SALIDA: ¡Guau! ¡Miau! ...ruido...
EXPLICACIÓN:
• a1 es "Animal" pero apunta a un Perro
• Java detecta en EJECUCIÓN que es Perro → ejecuta hacerRuido() de Perro
• Esto se llama ENLAZADO DINÁMICO
• Es la base del polimorfismo
3. CLASE ABSTRACTA
TEORÍA: Una clase abstracta NO se puede instanciar con new. Puede tener métodos abstractos (sin cuerpo, obligatorios) y métodos normales (con cuerpo, heredados).
// Clase abstracta - NO se puede hacer "new Figura()" public abstract class Figura { protected String color; public Figura(String color) { this.color = color; } // Método ABSTRACTO - las subclasses DEBEN implementarlo public abstract double calcularArea(); // Método NORMAL - se hereda directamente public void mostrarColor() { System.out.println("Color: " + color); } } public class Rectangulo extends Figura { private double base, altura; public Rectangulo(String color, double base, double altura) { super(color); // Llama al constructor del padre this.base = base; this.altura = altura; } @Override public double calcularArea() { return base * altura; } } public class Circulo extends Figura { private double radio; public Circulo(String color, double radio) { super(color); this.radio = radio; } @Override public double calcularArea() { return Math.PI * radio * radio; } } public class Main { public static void main(String[] args) { // ❌ NO se puede: Figura f = new Figura(); Figura r = new Rectangulo("Rojo", 5, 3); Figura c = new Circulo("Azul", 2); r.mostrarColor(); // Método del padre System.out.println(r.calcularArea()); // 15.0 System.out.println(c.calcularArea()); // 12.56... } }
SALIDA: Color: Rojo 15.0 12.566370614359172
EXPLICACIÓN:
• Figura es abstracta → no se puede instanciar
• calcularArea() es abstracto → cada subclase lo implementa diferente
• mostrarColor() es normal → se hereda sin cambios
• Cada figura calcula su área de forma distinta (POLIMORFISMO)
4. INTERFAZ - Implementación
TEORÍA: Una interfaz define un CONTRATO. Las clases que la implementan DEBEN tener todos sus métodos. No tiene código (normalmente).
// INTERFAZ - Define el contrato public interface Volador { void volar(); // Sin implementación boolean estaEnAire(); // Sin implementación default void info() { // Java 8+: método con código System.out.println("Puedo volar"); } } // Clase que firma el contrato public class Pajaro implements Volador { private boolean enAire = false; @Override public void volar() { enAire = true; System.out.println("El pájaro vuela"); } @Override public boolean estaEnAire() { return enAire; } } public class Avion implements Volador { private boolean enAire = false; @Override public void volar() { enAire = true; System.out.println("El avión despega"); } @Override public boolean estaEnAire() { return enAire; } } public class Main { public static void main(String[] args) { // Polymorfismo con interfaces Volador v1 = new Pajaro(); Volador v2 = new Avion(); v1.volar(); // El pájaro vuela v2.volar(); // El avión despega // También puedes hacer arrays de Volador Volador[] voladores = {new Pajaro(), new Avion()}; for (Volador v : voladores) { v.volar(); } } }
SALIDA: El pájaro vuela El avión despega El pájaro vuela El avión despega
EXPLICACIÓN:
• implements Volador = "me comprometo a implementar volar() y estaEnAire()"
• NO hay herencia, es un CONTRATO
• Puedes tratar Pajaro y Avion como "cosas que vuelan"
• Un clase puede implementar VARIAS interfaces
5. ARRAYLIST - Métodos principales
TEORÍA: ArrayList es una colección dinámica que puede crecer y decrecer. A diferencia de los arrays normales, no tiene tamaño fijo.
import java.util.ArrayList; public class Main { public static void main(String[] args) { ArrayList frutas = new ArrayList<>(); // AÑADIR elementos frutas.add("Manzana"); // índice 0 frutas.add("Banana"); // índice 1 frutas.add("Cereza"); // índice 2 frutas.add(1, "Naranja"); // inserta en índice 1 System.out.println("ArrayList: " + frutas); System.out.println("Tamaño: " + frutas.size()); // ACCEDER elementos System.out.println("Índice 0: " + frutas.get(0)); System.out.println("Índice 2: " + frutas.get(2)); // MODIFICAR frutas.set(2, "Fresa"); System.out.println("Modificado: " + frutas); // ELIMINAR frutas.remove(1); // elimina por índice frutas.remove("Manzana"); // elimina por objeto System.out.println("Después de eliminar: " + frutas); // BUSCAR System.out.println("¿Contiene Fresa? " + frutas.contains("Fresa")); System.out.println("Índice de Fresa: " + frutas.indexOf("Fresa")); // RECORRER System.out.println("\nCon for-each:"); for (String f : frutas) { System.out.println("- " + f); } // CON OBJETOS ArrayList numeros = new ArrayList<>(); numeros.add(10); numeros.add(20); numeros.add(30); System.out.println("\nSuma: " + (numeros.get(0) + numeros.get(1) + numeros.get(2))); } }
SALIDA: ArrayList: [Manzana, Naranja, Banana, Cereza] Tamaño: 4 Índice 0: Manzana Índice 2: Banana Modificado: [Manzana, Naranja, Fresa, Cereza] Después de eliminar: [Manzana, Fresa, Cereza] ¿Contiene Fresa? true Índice de Fresa: 1 Con for-each: - Manzana - Fresa - Cereza Suma: 60
MÉTODOS CLAVE:
• add(elem) → añadir al final
• add(idx, elem) → insertar en posición
• get(idx) → obtener elemento
• set(idx, elem) → modificar
• remove(idx/elem) → eliminar
• size() → tamaño
• contains(elem) → buscar
• clear() → vaciar
6. HASHMAP - Clave/Valor
TEORÍA: HashMap almacena pares CLAVE→VALOR. La búsqueda es MUY rápida porque usa una tabla hash.
import java.util.HashMap; public class Main { public static void main(String[] args) { // CLAVE (String) → VALOR (Integer) HashMap edades = new HashMap<>(); // AÑADIR edades.put("Ana", 25); edades.put("Luis", 30); edades.put("María", 28); // Si pones otra clave igual, SOBRESCRIBE el valor edades.put("Ana", 26); System.out.println("HashMap: " + edades); System.out.println("Tamaño: " + edades.size()); // OBTENER valor por clave System.out.println("Edad de Ana: " + edades.get("Ana")); System.out.println("Edad de Pedro (no existe): " + edades.get("Pedro")); // null // VERIFICAR existencia System.out.println("¿Existe Ana? " + edades.containsKey("Ana")); System.out.println("¿Existe valor 30? " + edades.containsValue(30)); // ELIMINAR edades.remove("Luis"); System.out.println("Después de eliminar: " + edades); // RECORRER System.out.println("\n--- Recorridos ---"); // Solo claves for (String clave : edades.keySet()) { System.out.println("Clave: " + clave); } // Solo valores for (Integer valor : edades.values()) { System.out.println("Valor: " + valor); } // Clave y valor for (String clave : edades.keySet()) { System.out.println(clave + " tiene " + edades.get(clave) + " años"); } // CON OBJETOS HashMap capitales = new HashMap<>(); capitales.put("España", "Madrid"); capitales.put("Francia", "París"); System.out.println("\nCapital de España: " + capitales.get("España")); } }
SALIDA: HashMap: {Ana=26, Luis=30, María=28} Tamaño: 3 Edad de Ana: 26 Edad de Pedro (no existe): null ¿Existe Ana? true ¿Existe valor 30? true Después de eliminar: {Ana=26, María=28} --- Recorridos --- Clave: Ana Clave: María Valor: 26 Valor: 28 Ana tiene 26 años María tiene 28 años Capital de España: Madrid
MÉTODOS CLAVE:
• put(clave, valor) → añadir/modificar
• get(clave) → obtener valor
• remove(clave) → eliminar
• containsKey(clave) → verificar clave
• containsValue(valor) → verificar valor
• keySet() → conjunto de claves
• values() → conjunto de valores
• Búsqueda O(1) - constante
7. HASHSET - Elementos únicos
TEORÍA: HashSet NO permite elementos duplicados. Si intentas añadir un elemento que ya existe, simplemente no lo añade.
import java.util.HashSet; public class Main { public static void main(String[] args) { HashSet colores = new HashSet<>(); // AÑADIR colores.add("Rojo"); colores.add("Azul"); colores.add("Verde"); colores.add("Rojo"); // Duplicado - no se añade colores.add("Amarillo"); System.out.println("HashSet: " + colores); System.out.println("Tamaño: " + colores.size()); // 4, no 5! // VERIFICAR existencia System.out.println("¿Contiene Rojo? " + colores.contains("Rojo")); System.out.println("¿Contiene Naranja? " + colores.contains("Naranja")); // ELIMINAR colores.remove("Azul"); System.out.println("Después de eliminar: " + colores); // RECORRER (sin orden) System.out.println("\nRecorriendo:"); for (String color : colores) { System.out.println("- " + color); } // CON NÚMEROS HashSet numeros = new HashSet<>(); numeros.add(1); numeros.add(2); numeros.add(3); numeros.add(2); // Duplicado numeros.add(3); // Duplicado System.out.println("\nNúmeros: " + numeros); System.out.println("Tamaño: " + numeros.size()); // 3 // OPERACIONES DE CONJUNTOS HashSet set1 = new HashSet<>(); set1.add("A"); set1.add("B"); set1.add("C"); HashSet set2 = new HashSet<>(); set2.add("B"); set2.add("C"); set2.add("D"); // Intersección HashSet copia = new HashSet<>(set1); copia.retainAll(set2); // Solo los comunes System.out.println("\nIntersección: " + copia); // [B, C] // Unión copia = new HashSet<>(set1); copia.addAll(set2); // Todos System.out.println("Unión: " + copia); // [A, B, C, D] } }
SALIDA: HashSet: [Rojo, Azul, Verde, Amarillo] Tamaño: 4 ¿Contiene Rojo? true ¿Contiene Naranja? false Después de eliminar: [Rojo, Verde, Amarillo] Recorriendo: - Rojo - Verde - Amarillo Números: [1, 2, 3] Tamaño: 3 Intersección: [B, C] Unión: [A, B, C, D]
CARACTERÍSTICAS:
• NO permite duplicados
• NO mantiene orden
• add() → returns true si se añadió, false si ya existía
• addAll(set) → unión
• retainAll(set) → intersección
• removeAll(set) → diferencia
• Útil para eliminar duplicados rápidamente
8. COMPARACIÓN: ArrayList vs HashMap vs HashSet
TEORÍA: Elige la estructura correcta según tu necesidad.
import java.util.*; public class Main { public static void main(String[] args) { // ARRAYLIST - Lista ORDENADA con índice System.out.println("=== ARRAYLIST ==="); ArrayList lista = new ArrayList<>(); lista.add("Perro"); lista.add("Gato"); lista.add("Pájaro"); System.out.println("get(1): " + lista.get(1)); // Por índice System.out.println("contains: " + lista.contains("Gato")); // Sí System.out.println("Orden: " + lista); // [Perro, Gato, Pájaro] // HASHSET - Elementos ÚNICOS sin orden System.out.println("\n=== HASHSET ==="); HashSet conjunto = new HashSet<>(); conjunto.add("Perro"); conjunto.add("Gato"); conjunto.add("Gato"); // Ignorado conjunto.add("Pájaro"); System.out.println("No hay get() por índice"); System.out.println("contains: " + conjunto.contains("Gato")); // Sí System.out.println("Orden: " + conjunto); // Sin orden garantizado // HASHMAP - Clave → Valor System.out.println("\n=== HASHMAP ==="); HashMap mapa = new HashMap<>(); mapa.put("Perro", 5); mapa.put("Gato", 3); mapa.put("Pájaro", 10); System.out.println("get(Clave): " + mapa.get("Gato")); // Por clave System.out.println("containsKey: " + mapa.containsKey("Gato")); // Sí System.out.println("Claves: " + mapa.keySet()); // [Perro, Gato, Pájaro] // ¿CUÁNDO USAR CADA UNO? System.out.println("\n=== ¿CUÁNDO USAR? ==="); System.out.println("ArrayList: Cuando NECESITAS ORDEN o ACCESO POR ÍNDICE"); System.out.println("HashSet: Cuando NECESITAS ELEMENTOS ÚNICOS"); System.out.println("HashMap: Cuando NECESITAS BÚSQUEDA POR CLAVE"); // EJEMPLO PRÁCTICO System.out.println("\n=== EJEMPLO PRÁCTICO ==="); // ArrayList de animales ArrayList animales = new ArrayList<>(); animales.add("León"); animales.add("Tigre"); // HashSet de especies únicas HashSet especies = new HashSet<>(); animales.forEach(a -> especies.add(a.split(" ")[0])); // Extrae "León", "Tigre" // HashMap para datos de cada animal HashMap peso = new HashMap<>(); peso.put("León", 190); peso.put("Tigre", 220); System.out.println("Especies: " + especies); System.out.println("Peso del León: " + peso.get("León") + " kg"); } }
SALIDA: === ARRAYLIST === get(1): Gato contains: true Orden: [Perro, Gato, Pájaro] === HASHSET === No hay get() por índice contains: true Orden: [Rojo, Verde, Amarillo] === HASHMAP === get(Clave): 3 containsKey: true Claves: [Perro, Gato, Pájaro] === ¿CUÁNDO USAR? === ArrayList: Cuando NECESITAS ORDEN o ACCESO POR ÍNDICE HashSet: Cuando NECESITAS ELEMENTOS ÚNICOS HashMap: Cuando NECESITAS BÚSQUEDA POR CLAVE === EJEMPLO PRÁCTICO === Especies: [León, Tigre] Peso del León: 190 kg
RESUMEN:
ArrayList → Lista ordenada, acceso por índice
HashSet → Elementos únicos, sin duplicados
HashMap → Búsqueda rápida por clave

COMPLEJIDAD:
ArrayList: búsqueda O(n), acceso O(1)
HashSet/HashMap: búsqueda O(1) constante
9. POLIMORFISMO con Interfaces
TEORÍA: Puedes declarar una variable del tipo de la interfaz y asignarle cualquier objeto que la implemente.
interface Comparable { int comparar(Object otro); } class Persona implements Comparable { String nombre; int edad; Persona(String nombre, int edad) { this.nombre = nombre; this.edad = edad; } @Override public int comparar(Object otro) { Persona p = (Persona) otro; return this.edad - p.edad; // Compara por edad } } class Main { public static void main(String[] args) { Persona p1 = new Persona("Ana", 25); Persona p2 = new Persona("Luis", 30); // Variable del tipo INTERFAZ Comparable c1 = p1; Comparable c2 = p2; // Pero solo puedes usar métodos de la interfaz int resultado = c1.comparar(c2); System.out.println("Resultado: " + resultado); // -5 // Arrays polimórficos Comparable[] personas = { new Persona("Ana", 25), new Persona("Luis", 30), new Persona("María", 20) }; // Ordenar usando la interfaz // Arrays.sort(personas); for (Comparable p : personas) { Persona pp = (Persona) p; // Cast para acceder a nombre System.out.println(((Persona)p).nombre); } } }
EXPLICACIÓN:
• Comparable c1 = p1; → la variable es del tipo interfaz
• Solo puedes usar métodos de la interfaz
• El objeto real puede ser cualquier clase que la implemente
• Esto permite escribir código GENÉRICO
10. CLASE ABSTRACTA + INTERFAZ JUNTAS
TEORÍA: Una clase puede HEREDAR de una abstracta E IMPLEMENTAR interfaces. Ejemplo: class Perro extends Animal implements Mascota, Identificable
// INTERFAZ 1 interface Volador { void volar(); } // INTERFAZ 2 interface Nadador { void nadar(); } // CLASE ABSTRACTA abstract class Animal { protected String nombre; public Animal(String nombre) { this.nombre = nombre; } abstract void comer(); } // CLASE que usa TODO class Delfin extends Animal implements Nadador { public Delfin(String nombre) { super(nombre); } @Override public void comer() { System.out.println(nombre + " come peces"); } @Override public void nadar() { System.out.println(nombre + " nada rápidamente"); } } class Pato extends Animal implements Volador, Nadador { public Pato(String nombre) { super(nombre); } @Override public void comer() { System.out.println(nombre + " come grano"); } @Override public void volar() { System.out.println(nombre + " vuela"); } @Override public void nadar() { System.out.println(nombre + " nada"); } } class Main { public static void main(String[] args) { Delfin d = new Delfin("Flipper"); Pato p = new Pato("Donald"); d.comer(); // come peces d.nadar(); // nada rápidamente p.comer(); // come grano p.volar(); // vuela p.nadar(); // nada // POLIMORFISMO Animal[] animales = {d, p}; for (Animal a : animales) { a.comer(); // Cada uno come a su manera } } }
SALIDA: Flipper come peces Flipper nada rápidamente Donald come grano Donald vuela Donald nada Flipper come peces Donald come grano
EXPLICACIÓN:
• extends Animal implements Volador, Nadador
• Una clase puede implementar VARIAS interfaces
• Solo puede extender UNA clase (abstracta o no)
• El Pato puede VOLAR y NADAR (2 interfaces)
• El Delfín solo puede NADAR (1 interfaz)
• Cada uno come() diferente → polimorfismo
